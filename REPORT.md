## 任务 B：分析

### B1. PPO 策略与纳什均衡的距离

#### 定量分析
根据 exploitability 数据：
- PPO 的 exploitability = 0.248933
- 纳什均衡的 exploitability = 0（理论值）
- 距离度量: PPO 策略可被最优响应策略额外获得约 0.2489 的期望收益

#### 为什么自我对弈的 PPO 不保证收敛到纳什均衡？

**1. 优化目标根本不同**

PPO 优化的是：
```
max E[R | π_self, π_opponent_current]
```

纳什均衡要求：
```
max_π min_opponent E[R | π, opponent]
```

PPO 只关心"打败当前对手"，不关心"防御未来可能出现的最强对手"。

**2. 循环支配问题**

不完美信息博弈中存在策略循环：
- 策略 A（激进 bluff）> 策略 B（保守 fold）
- 策略 B（保守 fold）> 策略 C（中等 call）
- 策略 C（中等 call）> 策略 A（激进 bluff）

自我对弈会陷入：
```
A 出现 → 学会 B 来克制 A → 学会 C 来克制 B → 回到 A
```

这种振荡永远不会稳定到混合策略的纳什均衡。

**3. 信息不对称导致的"遗忘"**

在 Kuhn Poker 中：
- 玩家不知道对手的牌
- PPO 可能学会针对"对手现在 70% 概率 bluff"
- 但当对手调整到 30% bluff，之前的防御策略不再有用
- PPO 没有"记忆"机制保留历史策略，会遗忘

相比之下，CFR 通过**累积遗憾值**记住所有历史对抗，避免遗忘。

**4. 理论保证的缺失**

- **完美信息博弈**（如围棋）：自我对弈有收敛保证（Minimax 定理）
- **不完美信息博弈**：策略梯度方法**没有**收敛到纳什均衡的理论保证
- CFR 及其变体（CFR+, MCCFR）有理论收敛保证

**实验证据**:
我们观察到 PPO 训练曲线仍有明显振荡。最终 PPO exploitability 为 0.2489，低于随机策略的 0.4583，但相比 CFR 的 0.0001 仍高出约 2196 倍。这验证了理论分析：PPO 可以改善策略质量，但自我对弈 PPO 不保证收敛到纳什均衡。

### B2. 密集奖励的设计与问题

#### 当前奖励方案（稀疏奖励）

当前实现使用的是终局奖励：一局游戏过程中，中间动作不给额外奖励；只有游戏结束后，根据 OpenSpiel 返回的最终收益给玩家分配奖励。赢的一方获得正收益，输的一方获得负收益，如果存在平局则收益为 0。这个设计的优点是奖励信号和真实目标完全一致，也就是最终筹码收益；缺点是训练信号比较稀疏，agent 只有到终局后才知道前面的动作整体效果如何。

#### 提议的密集奖励方案

如果要改成密集奖励，我会保留终局筹码收益作为主要信号，同时给中间动作很小的辅助奖励。例如，下注或跟注可以得到少量正奖励，用来鼓励 agent 主动争取底池；过早 pass 或 fold 可以给轻微惩罚，避免策略过于保守；当 agent 拿到较强牌并选择下注时，也可以给少量正反馈，因为这通常更符合价值最大化。这个方案的目标不是替代终局输赢，而是让 agent 在终局前也能收到一些方向性信号。

不过，这种设计必须非常谨慎。扑克里的动作价值高度依赖隐藏信息和对手策略，弱牌下注有时是 bluff，强牌 check 有时是诱导对手下注；如果中间奖励设计得太强，就可能把人类直觉硬编码进策略，反而破坏博弈中的混合策略学习。

#### 这个方案会被"钻空子"的方式

**漏洞 1: 过度激进**

- **问题**: `BET` 动作给 +0.1 即时奖励
- **被钻空子的方式**: Agent 学会"无脑 Bet"，即使拿到最弱的牌
- **为什么会发生**: 
  - +0.1 × 多次 = 累积正奖励
  - 终局 -1 的惩罚被 γ^n 折扣弱化
  - 如果 γ=0.99, n=3, 终局 -1 只值 -0.97，抵不过 3 次 +0.1 = +0.3
- **后果**: 对手学会识别"无脑 Bet"并利用

**漏洞 2: 忽视真实目标**

- **问题**: Agent 优化"获得中间奖励"而非"赢得游戏"
- **被钻空子的方式**: 
  - 故意延长游戏以多次触发"信息获取"奖励
  - 在不该 Bet 的时候 Bet（为了 +0.1）
- **后果**: 训练时奖励很高，但真实评估时表现很差

**漏洞 3: 信息泄露陷阱**

- **问题**: "信息价值"奖励鼓励 agent 做出暴露信息的动作
- **被钻空子的方式**: Agent 学会做一些"获取信息"的动作，但这些动作同时泄露了自己的牌力
- **Poker 中的反例**: 在弱牌时 Bet（想"试探对手"），反而暴露自己在 bluff
- **后果**: 被对手利用，exploitability 提高

**漏洞 4: 短视的局部最优**

- **问题**: 密集奖励将长期博弈问题分解为短期优化
- **被钻空子的方式**: Agent 学会每一步都"看起来合理"，但整体策略糟糕
- **类比**: 国际象棋中，每一步都"吃子"看起来好，但可能牺牲了战略布局
- **后果**: 陷入局部最优，无法学到复杂的 bluffing 策略


#### 更好的密集奖励设计原则

1. **对齐性**: 任何提高中间奖励的行为必须也提高终局获胜概率
2. **避免偏见**: 不要手工编码"合理动作"（人类直觉在博弈中常常错）
3. **使用学习**: 用 value function 本身作为中间奖励（这就是 TD learning）
4. **稀疏是特征**: 在短 horizon 博弈中，稀疏奖励是最"诚实"的信号

#### 建议

**在 Kuhn Poker 中，不应该使用密集奖励，原因：**

1. **Horizon 太短**: 只有 2-3 步，信用分配不是问题
2. **博弈的特殊性**: 唯一客观的信号就是"谁赢了"，中间没有"对错"
3. **已有解决方案**: Value function bootstrapping（TD(λ)）已经提供了隐式的中间奖励

如果一定要用，唯一合理的方式是：
```python
reward_t = γ * V(s_{t+1}) - V(s_t)  # TD error
```
即用学习到的 value function 自己做 bootstrapping，而非人工规则。

### B3. 真实遇到的失败/异常

#### 失败案例：评估进度条卡在 0/1000

**现象**:

训练能正常推进，但到第一次评估时卡住：

```text
Training:  10%|...| 497/5000 [...]
Evaluating Player 0:   0%|          | 0/1000 [00:00<?, ?it/s]
```

一开始看起来像是评估特别慢，或者 `tqdm` 没刷新。但训练阶段每秒能跑几百局，Kuhn Poker 单局又极短，所以评估 1000 局不应该长时间停在 0。

**定位过程**:

1. **缩小范围**: 单独运行 `evaluate_against_random(game, agent, 0, num_games=10)`，发现函数不返回，说明问题在评估循环内部。
2. **对照训练代码**: 训练函数 `play_episode_self_play()` 在 chance node 抽样后会调用 `state.apply_action(action)`。
3. **检查评估代码**: `evaluate_against_random()` 里 chance node 只做了：
   ```python
   outcomes, probs = zip(*state.chance_outcomes())
   action = np.random.choice(outcomes, p=probs)
   ```
   但漏掉了 `state.apply_action(action)`。
4. **根因分析**:
   - Kuhn Poker 的初始状态是 chance node，也就是先发私牌
   - 评估代码抽到了发牌动作，但没有把动作应用到环境
   - 下一轮 while 循环仍然停在同一个 chance node
   - 所以第一局游戏永远无法结束，外层 `for 1000 games` 连第一步都走不完

**解决方案**:

在评估函数的 chance node 分支里补上状态推进：

```python
if state.is_chance_node():
    outcomes, probs = zip(*state.chance_outcomes())
    action = np.random.choice(outcomes, p=probs)
    state.apply_action(action)
else:
    ...
```

修复后，`Evaluating Player 0` 能正常从 0/1000 跑到结束，完整训练也能继续进入后续 episode。

**经验教训**:

1. **环境循环必须保证每次迭代都会推进状态**，否则 `while not state.is_terminal()` 很容易变成死循环。
2. **训练路径和评估路径要分别测试**。训练能跑通不代表评估也能跑通，因为两条路径虽然相似，但不是同一段代码。
3. **遇到进度条卡住时，先用很小的 `num_games=1/10` 复现**，比直接看完整训练日志更快定位。
4. **OpenSpiel 的 chance node 也是环境状态的一部分**，抽样只是选动作，必须显式 `apply_action` 才会进入玩家决策节点。

## 任务 C：对手建模小实验

**实现**: `src/task_c_opponent_modeling.py`

选择了任务 C 的第三项：观察对手策略固定/变化时，训练好的 PPO agent 表现如何变化。实验不重新训练 PPO，而是加载 A2 训练得到的模型，分别让 PPO 面对几类手写对手策略：

1. **随机策略**: 每个合法动作均匀随机选择。
2. **保守策略**: 总是 pass/check/fold。
3. **激进策略**: 总是 bet/call。
4. **简单牌力策略**: K 总是 bet/call，J 总是 pass/fold，Q 面对下注时 call，否则 check。
5. **变化策略**: 每局随机切换上述一种对手风格。

每种对手策略下，分别评估 Player 0 PPO 和 Player 1 PPO，各 5,000 局。

| 对手策略 | Player 0 平均收益 | Player 0 胜率 | Player 1 平均收益 | Player 1 胜率 |
|----------|-------------------|---------------|-------------------|---------------|
| 随机 | +0.0236 | 42.38% | +0.1310 | 49.84% |
| 保守 | -0.0200 | 49.00% | +0.6424 | 82.12% |
| 激进 | +0.0090 | 33.64% | -0.0930 | 32.20% |
| 简单牌力 | -0.0238 | 48.80% | -0.0472 | 49.50% |
| 每局切换 | +0.0156 | 44.36% | +0.0900 | 49.58% |

**观察**:

- PPO 对随机和变化对手仍能保持正收益，说明它学到了一些泛化的策略结构。
- Player 1 对保守对手收益很高，说明后手 PPO 很会利用“不下注/容易弃牌”的对手。
- 面对简单牌力策略时两个位置收益都接近 0 或略负，说明基于牌力的固定策略已经能削弱 PPO 的优势。
- 面对总是 bet/call 的激进对手，Player 1 表现明显变差，说明 PPO 对极端下注频率的鲁棒性不足。
